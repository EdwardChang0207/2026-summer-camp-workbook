# 下午實作｜文字 RPG《魔法地下城》— 老師 Demo 說明

> 本資料夾是**下午實作課的教師示範版（解答）**。學生**不會**拿到這些 `.py`，他們要在 Colab 上「自己一行一行打出來」；老師用本 Demo 對照進度、預先熟悉成品、現場除錯。

---

## 一、這份 Demo 是什麼

四天的下午實作是**同一支遊戲、逐日長大**：每天把上午剛學的觀念，立刻變成遊戲的一個新功能。第四天結束時，每位學生手上都會有一支**從頭玩到尾、能打三場 BOSS** 的文字 RPG，正好當作成果發表的作品。

| 檔案 | 對應日 | 新加入的觀念 | 遊戲長出什麼 |
|------|--------|--------------|--------------|
| `day1_角色建立與開場.py` | Day 1 | `print` / 變數 / `input` / `int()` / f-string | 開場劇情＋輸入姓名等級＋角色卡 |
| `day2_職業選擇與分支.py` | Day 2 | 比較 / `if`-`elif`-`else` / `and`·`or` | 選職業、走岔路，劇情會分歧 |
| `day3_回合制戰鬥.py` | Day 3 | `for`+`range` / `while` / `break` / `continue` | 和哥布林進行多回合戰鬥 |
| `day4_完整RPG成果.py` | Day 4 | `def` / `list`+`append` / `random` | 函式封裝＋多場 BOSS＋背包掉寶＝完整遊戲 |

> 對應官網下午專案設定：Day1 角色選擇介面 → Day2 職業判斷 → Day3 回合制戰鬥 → Day4 成果整合，與本 Demo 完全一致。

## 二、怎麼執行（兩種方式）

**A. Google Colab（學生上課用）**
1. 開新筆記本 → 把程式碼貼進一個 cell。
2. 按左側 ▶ 播放鍵執行。
3. `input()` 會在 cell 下方跳出輸入框，打字後按 Enter。

**B. 終端機（老師自己測試用）**
```bash
python3 day1_角色建立與開場.py
```

> ⚠️ Day4 用到 `random`，每次傷害數字都不一樣，這是正常的（亂數）。輸出長度會因戰況而異。

## 三、每日執行進度與教學節奏

### Day 1 — 角色建立與開場
- **今日目標**：玩家輸入姓名與等級，程式印出一張 f-string 角色卡。
- **完成判準**：能跑出「姓名／等級／生命值／攻擊力」四行，且 HP、攻擊力是用變數**算**出來的（不是寫死）。
- **帶法**：先一起完成「開場 print → 問名字 → 印角色卡」，再放手讓學生改成自己的角色設定（屬性公式、開場台詞）。
- **常見卡點**：
  - 忘了 `int()` → 等級拿來乘會變成字串重複（`"5"*20`）或報錯。
  - f-string 忘了開頭 `f` → 直接印出 `{name}` 字面。

### Day 2 — 職業選擇與劇情分支
- **今日目標**：用 `if/elif/else` 讓玩家「選職業」「走岔路」，不同選擇給不同屬性與台詞。
- **完成判準**：輸入 1/2/3 會得到不同職業；輸入 左/右 劇情不同；最後用 `and`/`or` 給一句狀態評語。
- **帶法**：強調**縮排**＝「這段屬於哪個 if」。建議先做職業三選一，再加岔路，最後加 `and/or` 綜合判斷。
- **常見卡點**：
  - 把 `=`（賦值）當成 `==`（比較）。
  - 縮排沒對齊 → `IndentationError`。
  - 比較字串時大小寫或全半形不一致（例如 `"左"` vs 空白）。

### Day 3 — 回合制戰鬥
- **今日目標**：用 `while` 讓戰鬥一回合接一回合，直到一方 HP 歸零。
- **完成判準**：能打贏哥布林、回合數會增加；`break`（敵人倒下立刻結束）、`continue`（喝藥水跳過攻擊）都有作用。
- **帶法**：先畫流程圖——「兩邊都活著就重複：玩家攻擊 → 敵人還活著就反擊」。`while` 的結束條件是重點。開場用 `for i in range(3,0,-1)` 做戰鬥倒數，順帶複習 for。
- **常見卡點**：
  - **無限迴圈**：忘了在迴圈內減 HP，或 `turn` 沒增加。提醒：「迴圈裡一定要有東西讓條件愈來愈接近結束。」
  - `while` 條件用錯 `and`/`or`（應是「兩邊都活著」＝ `and`）。

### Day 4 — 完整 RPG 成果
- **今日目標**：用 `def` 把戰鬥包成函式、用 `list` 管理敵人與背包、用 `random` 做亂數傷害與掉寶，組成完整遊戲。
- **完成判準**：能依序打哥布林 → 骸骨戰士 → 火龍王三場，HP 會延續、背包會用 `append` 累積、結尾用 `len` 統計戰利品。
- **帶法**：上午剛學 def/list/random，下午就把 Day3 的戰鬥**抽成 `battle()` 函式**，再用 `for` 跑敵人清單呼叫它——讓學生親眼看到「函式 = 可重複使用的積木」。**程度落差大時**，本檔可拆成「基本款（固定傷害、單場戰鬥）」與「進階款（亂數＋多場＋背包）」兩段，讓快的同學往進階做。
- **常見卡點**：
  - 函式定義了卻忘了**呼叫**。
  - `return` 的值沒接住（例如忘了 `hero_hp = battle(...)`，HP 不會延續）。
  - `random.randint(a, b)` 的 a 比 b 大（攻擊力太低時 `atk-5` 變負）。

## 四、給老師的三個提醒

1. **不要直接把解答貼給學生**（頑扣教學鐵則：不洩答）。卡關時用引導提問：「你想讓它重複，該用哪個迴圈？」「這個變數現在的值是多少？」
2. **一次一觀念**：每加一個新功能就先跑一次看結果，再加下一段，避免一次貼一大段全錯找不到問題。
3. **鼓勵客製化**：屬性公式、職業、敵人名字、台詞都可以讓學生自由改——成果發表時最有成就感的就是「這是我自己的遊戲」。
