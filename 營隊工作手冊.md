## 3. 使用環境與帳號

本營隊**線上進行**，共使用三個環境：視訊平台（上課主頻道）、官網（上午教學）、Colab（下午實作）。

### 3-0 視訊平台（上課主頻道）
- **平台**：【Google Meet】
- **老師必做**：**螢幕分享**官網或 Colab 畫面給學生看，讓學生跟著操作；學生各自開自己的視窗操作，不共用一個畫面，授課時請記得錄影。
- **建議設定**：
  - 請學生開**鏡頭**（有助於老師判斷狀態）、麥克風靜音，需發言時再開。
  - 使用**聊天室**傳連結、貼錯誤訊息、讓學生快速回應（例如打「好」確認看到）。
  - 老師可不定時請學生在聊天室回覆「好」或貼出目前的程式碼，確認跟上進度。

### 3-1 官網（上午授課）
- **入口連結**：`https://www.letsfunnycoding.com/python-summer-camp?mode=lobby`
- **進入流程**：登入學生帳號 → 進「大廳」→ 點 **🏛️ 學院** → 選當日 Day → 跟著章節上課。
  - `mode=lobby`＝大廳（選學院/地下城）；`mode=academy`＝直接進學院；`mode=dungeon`＝直接進地下城。
- **學院 vs 地下城**：學院＝照章節教學（上午用這個）；地下城＝同觀念的闖關練習（給快的同學挑戰）。地下城需先完成當日學院章節才解鎖。
- **程式在哪跑**：官網章節內建程式執行（瀏覽器即可跑），學生**不需另外安裝**。


### 3-2 Google Colab（下午實作）
- **入口連結**：`https://colab.research.google.com/`
- **用途**：下午的文字 RPG 專案在 Colab 寫與執行。
- **基本操作**：新增筆記本 → 程式碼貼/打在 cell → 按 ▶ 執行 → `input()` 會在下方跳出輸入框。

#### ⚠️ Colab 帳號問題處理 SOP（務必照走）
有些孩子登入的是**公司或學校的 Google 帳戶**，這類帳戶常被組織政策**禁用 Colab**（會出現無法開啟 / 權限不足）。處理流程：

1. **確認是否為公司／學校帳戶造成**
2. **詢問學生**：是否有其他 Google 帳戶（例如個人 Gmail）？
   - **有** → 改用該帳戶登入 Colab（多數可立即解決）
   - **沒有** → 透過 LINE 回報行政窗口 → 行政傳訊家長協助提供可用帳戶

- **老師現場**：先試「換個人帳戶」這條路，多數可立即解決。
- **真的沒有可用帳戶**：**不要卡在那個學生身上拖住全班**——用老師示範暫時跟上，同時 LINE 回報行政，由行政聯繫家長。
- 回報時請附：學生姓名、卡住的畫面（截圖佳）、已嘗試的步驟。（回報範本見 §7）

> 💡 **預防**：第一天上午結束前，可先請全班「試開一次 Colab」，把帳戶問題在下午正式用之前先篩出來、提早處理。

## 6. 教學原則與帶班心法

1. **不洩答**：卡關用提問引導（「你想讓它重複，該用哪個？」「這個變數現在值多少？」），不直接給答案。
2. **一次一觀念**：每加一個新功能先跑一次看結果，再加下一段。
3. **先看到成品**：每段開始前先展示「等等會做出什麼」，建立動機。
4. **巡堂優先**：講解精簡、把時間留給巡堂；同一個錯出現 3 次以上就全班暫停統一講。
5. **鼓勵客製化**：屬性、職業、敵人、台詞讓學生自由改，成果發表最有成就感的是「這是我自己的遊戲」。
6. **照顧落差**：快的給地下城 / 進階款；慢的降低目標、確保「能跑出東西」的成就感。
7. **正向回饋**：多稱讚「你自己 debug 出來了！」勝過稱讚「你好聰明」。

## 7. 注意事項與突發狀況處理

### 7-1 技術類

| 狀況 | 現場處理 |
|------|----------|
| **學生進不去官網課程** | 確認帳號是否登入正確；若帳密無誤仍無法進入，立即回報行政處理。 |
| **Colab 打不開 / 權限不足** | 走 §3-2 帳號 SOP：先換個人帳戶，不行就 LINE 回報行政→家長。不要卡住全班。 |
| **程式「卡住一直轉圈」（無限迴圈）** | Day3 最常見。按 cell 的「停止 ⏹」→ 檢查「迴圈裡有沒有讓條件改變的那一行」。 |
| **程式紅字報錯** | 帶學生看**最後一行**錯誤訊息：`SyntaxError`（標點/引號）、`IndentationError`（縮排）、`NameError`（變數名打錯/沒定義）、`TypeError`（型別，常是忘了 int()）。 |
| **學生網路突然斷線** | 先等 1–2 分鐘觀察是否自動重連；若未回，繼續上課，由行政 LINE 聯繫家長確認；回來後請學生從聊天室補看老師分享的進度。 |
| **老師自己斷線 / 視訊中斷** | 立刻重連；重連前在聊天室留言「請稍等，我重連中」；若超過 3 分鐘無法重連，以手機進入會議繼續，同時通報行政。 |
| **螢幕分享失效 / 學生看不到畫面** | 停止分享後重新開始；確認分享的是「特定視窗」非「桌面整體」；備案：把官網連結貼聊天室，讓學生自行開啟跟著走。 |
| **全班網路中斷（平台崩潰）** | 先等 5 分鐘；若無法恢復，切換備用平台（LINE 群組語音 / 電話通知）；備案：老師錄製短片或截圖傳聊天室，讓學生自行練習。 |
| **學生程式碼不見了** | 提醒隨手存檔；Colab 可用「檔案→在雲端硬碟儲存副本」；下課前提醒全班存一次。 |

> 📌 共通原則：**任何技術狀況超過 5 分鐘無法解決，就先讓該生透過老師螢幕分享跟上，再回報行政**，不要讓全班停擺。

### 7-2 教學類

| 狀況 | 處理 |
|------|------|
| **學生卡關放棄** | 拆小目標（先求印出一行就好），給一次小成功再往下；用提問引導而非給答案。 |
| **進度落差大** | 設「基本線」全班都要到，快的給延伸（地下城 / 進階款）；慢的不催，確保有成就感。 |
| **學生說「好無聊 / 我會了」** | 丟挑戰：客製化遊戲、地下城 BOSS、幫鄰座除錯（小老師）。 |
| **吵鬧 / 分心 / 玩別的網站** | 重新聚焦：給明確小任務 + 限時；直接點名請該生回答問題或展示畫面。 |
| **吵鬧干擾課程（麥克風）** | 直接在 Google Meet 將該生**靜音**；告知學生需發言時舉手再開麥克風。 |
| **屢次吵鬧 / 聊天室刷頻** | 警告無效後，**將該生請出 Meet**，立即通報行政，由行政與家長協調後續處理方式。 |
| **學生互相比較 / 嘲笑進度慢** | 強調「每個人節奏不同、會自己 debug 最厲害」，建立安全的學習氛圍。 |

### 7-3 行政 / 安全類

| 狀況 | 處理 |
|------|------|
| **學生未上線 / 遲到上線** | 點名後立即 LINE 回報行政，由行政聯繫家長；遲到學生進入後補上當下進度。 |
| **請假 / 中途離線 / 提前下課** | 記錄於出席表並回報行政；中途離線若未說明原因，由行政確認家長知情。 |
| **學生身體不適（家長反映）** | 線上課程無法直接處理，立即通知負責人，由行政聯繫家長；情況嚴重請家長帶就醫，當日課程讓學生事後補看錄影或筆記。 |
| **視訊中衝突 / 情緒問題** | 先用聊天室私訊穩定學生；若持續影響課程，將該生靜音後繼續上課；嚴重者交由負責人聯繫家長。 |
| **午休（12:00–13:30）** | 宣布下午 **13:30** 準時重回視訊，提醒學生確認網路與裝置；老師不需在線待命，但需在 13:25 前重新進入會議等待。 |
| **下課後（16:00）** | 視訊點名確認人數齊全，感謝今日參與，提前預告明日主題後結束會議；有特殊狀況請在結束前回報行政。 |

### 7-4 升級回報原則
- **能自己解的**（單一學生的程式錯誤、操作問題）→ 現場處理。
- **影響個別學生上課的**（帳號、Colab、進不去課程）→ LINE 回報行政，繼續上課。
- **影響全班 / 安全 / 健康的**（網路全斷、受傷、衝突）→ **立即**電話 / 當面通知負責人。

## 附錄：常見錯誤訊息速查（給學生看錯誤的口訣）
| 訊息 | 白話意思 | 先檢查 |
|------|----------|--------|
| `SyntaxError` | 文法寫錯了 | 引號、括號、冒號、全形標點 |
| `IndentationError` | 縮排不對 | 該縮排的有沒有縮、有沒有對齊 |
| `NameError` | 叫了一個不存在的名字 | 變數名是否打錯 / 是否還沒定義 |
| `TypeError` | 型別兜不起來 | 是不是忘了 `int()` 把字串轉數字 |
| `ValueError` | 值不對（如把"abc"轉 int） | 輸入內容是否符合預期 |
| 一直轉圈不停 | 無限迴圈 | 迴圈內有沒有讓條件趨近結束的那行 |
